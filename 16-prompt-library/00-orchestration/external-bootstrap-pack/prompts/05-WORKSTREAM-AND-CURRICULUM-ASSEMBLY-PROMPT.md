# Prompt 05 — Workstream and Curriculum Assembly

## Objective

Assemble the full KUTUMBA knowledge base from supplied operating documents without fabricating missing final content.

## Nine workstreams

Organize content into:

1. Governance, Charter and Temple Relationship
2. Three-Year Curriculum Architecture
3. First Six-Month Detailed Curriculum
4. Children and Youth Formation
5. Parent Formation and Family Care
6. Prasāda, Hospitality and Weekly Operations
7. Kīrtana, Worship, Prayers and Bhakti Laboratories
8. Festivals, Yātrās, Outdoor Activities and Calendar
9. Repository, Drive, Website and Digital Library

Also maintain:

- KUTUMBA Setu
- research and source register
- templates
- weekly program library
- parent library
- child library
- facilitator library

## Missing-workstream rule

If a final workstream document was not supplied:

- do not generate a fake “complete” operating model
- create a structured README
- preserve established decisions from the master model
- list expected deliverables
- create a gap record
- create a production prompt in `16-prompt-library`
- mark status as `SOURCE NOT YET SUPPLIED` or `DRAFT REQUIRED`

## Three-year design

Preserve the established architecture:

- Year 1 — Sambandha and family foundations
- Year 2 — Abhidheya and regulated practice
- Year 3 — Śāstric readiness, service and leadership

Create:

- curriculum map
- domain matrix
- prerequisite map
- source coverage map
- knowledge/skill/practice/character outcomes
- adult/child differentiation
- assessment and review model
- Bhakti Śāstrī readiness boundary
- initiation boundary
- 40-active-week annual structure
- cycle and off-week logic

Do not create shallow detailed lessons for all three years merely to fill folders.

## Detailed-week standard

Future detailed weekly lessons must match the quality and completeness of the first-six-month curriculum.

Create a formal lesson schema and master template with required metadata and required sections.

Create a production backlog by year, cycle, and week. Clearly distinguish:

- designed architecture
- detailed lesson complete
- detailed lesson draft
- source review required
- worship review required
- safety review required
- not started

## Weekly resource model

Create a consistent folder structure for each detailed week:

```text
week-code-title/
├── README.md
├── facilitator-guide.md
├── parent-lesson.md
├── children/
├── bhakti-lab.md
├── family-home-practice.md
├── worksheet.md
├── slide-outline.md
├── sources.yaml
└── review-status.yaml
```

Do not duplicate full content unnecessarily. Use a clear canonical file and derivative views when possible.

## Quality boundary

No advanced confidential topics for children without approval.

No invented worship procedures.

No public spiritual ranking.

No replacement of temple, guru, initiation, Bhakti-vṛkṣa, or Bhakti Śāstrī authorities.
