# Prompt 09 — Visuals, Diagrams, and Image Rights

## Objective

Use visuals because they improve understanding, not because every file needs decoration.

## Visual selection test

Create a visual only when it helps one of these tasks:

- compare;
- sequence;
- locate;
- classify;
- trace cause and effect;
- distinguish levels;
- remember;
- rehearse a process;
- tell a story.

## Preferred visual formats

1. Mermaid for flowcharts, timelines, concept maps, and simple relationships.
2. Original SVG for reusable diagrams.
3. Original PNG exports only when required.
4. Draw.io source when complex manual layout is necessary.
5. Rights-cleared photographs or devotional artwork.
6. AI-generated conceptual illustrations only with review and clear labeling.

## Do not

- create a mind map for every module;
- download random Google images;
- copy BBT paintings without permission;
- generate inaccurate Deity, ācārya, mantra, tilaka, or ritual imagery;
- use sensational death/rebirth images for children;
- treat visual beauty as authority.

## Per-module visual brief

For each chosen visual specify:

- learning problem;
- audience;
- visual type;
- exact content;
- what must be omitted;
- labels;
- accessibility alt text;
- source status;
- rights status;
- reviewer;
- Gamma placement;
- print use;
- child-safety concerns.

## Required utility outputs

Create scripts/specifications for:

- Mermaid syntax validation;
- missing alt-text detection;
- image-rights register validation;
- orphaned asset detection;
- oversized binary detection;
- SVG metadata checking.

## BBT/Prabhupāda artwork

Use metadata and external links until written permission or a clear licence is documented.

An image discovered through search remains `rights-status: unknown`.
