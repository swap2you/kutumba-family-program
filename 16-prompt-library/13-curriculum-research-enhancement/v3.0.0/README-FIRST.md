# KUTUMBA Curriculum Research and Enhancement Prompt Library v3.0

## Purpose

This pack upgrades the existing KUTUMBA repository from a structurally complete documentation repository into a source-grounded, pedagogically usable, visually supported curriculum knowledge base.

It is designed for the public repository:

`swap2you/kutumba-family-program`

Local root:

`C:\Development\Workspace\DevotionalRepo\kutumba-family-program`

The pack does **not** build an application. It expands and validates curriculum content, source traceability, age-banded lesson design, visual specifications, Gamma prompts, cycle projects, celebrations, and companion reading programs.

## Core operating principle

**Research depth behind the lesson; simplicity and rasa in the live session.**

A weekly/module folder may contain a deep research dossier, but the live Friday session must remain focused. Do not attempt to teach every discovered reference in one sitting.

## Current repository finding

The existing repository structure is strong. The first-six-month monolithic source is preserved and 18 derivative folders exist. However, several derivative component files are structurally present but substantively empty. For example, C1-W2 has empty analogy, materials, and newcomer-adaptation sections. The present validator verifies existence more strongly than semantic completeness.

This pack corrects that.

## Required age bands

Replace the old three-band delivery model with two operational groups:

- **Lāla–Lālī:** ages 4–8
- **Kiśora–Kiśorī:** ages 9–14

Within each group, use a core activity plus an extension/challenge so a four-year-old and an eight-year-old, or a nine-year-old and a fourteen-year-old, are not treated as developmentally identical.

## Run order

1. Read `00-START-HERE.md`.
2. Run `prompts/01-MASTER-ORCHESTRATOR.md`.
3. Cursor executes Prompts 02–13 in order.
4. Use Prompt 14 only after an interruption.
5. Run Prompt 15 in a fresh read-only review context.

## Keys and accounts

No API key is required for the first enhancement pass.

- Cursor may research public approved sources.
- NotebookLM is used manually as a source-grounded study assistant.
- Gamma may be used manually through Create with Agent, Paste, or Import.
- Do not place any credential in Git.
- Gamma API automation is optional later and requires a Gamma Pro-or-higher account. Any key must remain in a local ignored environment variable.

## Copyright boundary

Do not copy full Śrīla Prabhupāda translations or purports from VedaBase into the public repository. Store references, stable links, short excerpts, and original KUTUMBA summaries. Full reproduction requires documented permission from the rights holder.

Do not download images from Google merely because they appear in search. Google search is discovery, not a licence.
