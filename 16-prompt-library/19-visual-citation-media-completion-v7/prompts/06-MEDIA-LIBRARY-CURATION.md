# Prompt 06 — Media Library Curation

## Objective

Convert empty or generic media indexes into exact, governed module-level recommendations. Do not download the media.

## Priority scope

Fully curate:

- C1-W1 through C1-W6;
- C3-W2 because of the Kṛṣṇa identity and expansion visual requirement.

Create a backlog—not fabricated selections—for the remaining modules.

## Exact media record schema

Each candidate needs:

- media ID;
- module ID;
- speaker/creator;
- exact title;
- date;
- place;
- source institution/site;
- stable URL;
- media type;
- language;
- total duration if known;
- recommended excerpt start/end if verified;
- transcript link if available;
- one-paragraph relevance summary;
- exact claim or teaching supported;
- intended audience;
- age suitability;
- sensitive-content note;
- authority tier;
- rights posture;
- download/republication permission;
- link verification status;
- human doctrinal review;
- human pedagogy review;
- selected/rejected/deferred status;
- rejection reason where applicable.

## Sources

Use the governed public source catalog. Prioritize:

1. Śrīla Prabhupāda’s books through VedaBase;
2. VedaBase lectures, conversations and letters;
3. Bhaktivedanta Archives / PrabhupadaVani / Vanisource where exact records are verifiable;
4. official ISKCON education and congregation resources;
5. reviewed supplementary resources only as subordinate enrichment.

Do not use a YouTube channel merely because it is popular. Verify channel ownership, exact video and source lineage.

## Teacher media

`APPROVED-TEACHER-MEDIA-INDEX.md` must not say “approved” until a named human reviewer approves it.

Rename status fields where necessary:

- `candidate`
- `verified-candidate`
- `human-review-required`
- `approved-for-internal-pilot`
- `rejected`
- `deferred`

## Visual/media references

For copyrighted paintings/photos:

- catalog the source page;
- record rights as link-reference-only or permission-required;
- do not commit image files;
- add a permission request path when an asset is uniquely valuable.

## Education library use

Review the official ISKCON education catalog item-by-item for the seven currently mapped modules. Record the exact applicable activity pattern or teaching method. Do not claim a resource is used merely because its homepage is linked.

For the eleven deferred modules, explain whether:

- no resource is relevant;
- access failed;
- rights unclear;
- content not reviewed;
- original KUTUMBA pedagogy is preferable.

## Deliverables

- curated module media indexes for priority scope;
- updated source-to-module map;
- media selection policy;
- verification queue;
- `build-evidence/V7-MEDIA-CURATION-REPORT.md`;
- `build-evidence/V7-SOURCE-LIBRARY-UTILIZATION-REPORT.md`.

No media binaries are required.
