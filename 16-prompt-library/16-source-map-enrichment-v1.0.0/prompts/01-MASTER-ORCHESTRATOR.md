## Orchestration rules

1. Execute Prompts 02–11 in numeric order.
2. Read each prompt completely before acting.
3. Keep a phase ledger in:
   `build-evidence/SOURCE-MAP-ENRICHMENT-LEDGER.md`
4. After each phase:
   - run targeted validation;
   - record exact files changed;
   - record unresolved findings;
   - commit logically only when the phase is internally consistent.
5. Do not overwrite user-authored or human-reviewed content without preserving provenance and a correction record.
6. Do not create large content dumps.
7. Do not treat a successful HTTP response as doctrinal verification.
8. Do not treat a public webpage as redistribution permission.
9. Do not infer approval for speakers or institutions.
10. Stop only for a hard blocker:
    - authoritative source file missing;
    - merge conflict that risks losing user work;
    - repository not at the expected canonical root;
    - a requested action would require copying restricted content;
    - an identity cannot be resolved without guessing.
