# Prompt 07 — Build the Internal Prompt Library

## Objective

Place a reusable, version-controlled prompt library inside the repository so future content can be produced consistently without depending on one chat.

## Required folders

```text
16-prompt-library/
├── README.md
├── 00-orchestration/
├── 01-governance/

├── 09-digital-publishing/

```

## Prompt requirements

Every production prompt must specify:

- purpose
- source inputs
- governing decisions
- scope
- exclusions
- required output files
- required document structure
- citation rules
- copyright rules
- privacy rules
- doctrinal or worship-review gates
- quality checklist
- status update
- validation
- Git commit expectation

## Required prompts

Create prompts for:

- importing a new source document
- producing a complete operating model
- producing one detailed weekly lesson
- producing a six-week cycle
- generating age-banded child adaptations
- producing parent formation material
- producing a bhakti laboratory
- producing a festival/yātrā pack
- producing prasāda operations material
- producing KUTUMBA Setu material
- verifying citations
- reviewing safeguarding content
- reviewing worship content
- generating a family-facing derivative
- updating the publication manifest
- completing an independent repository audit
- remediating audit findings
- resuming interrupted work

## Prompt quality

Prompts must preserve the depth of existing KUTUMBA documents. They may not encourage generic summaries, bulk shallow generation, or invented authority.

Copy the external startup prompt pack into:

`16-prompt-library/00-orchestration/external-bootstrap-pack/`

Keep a version record.
