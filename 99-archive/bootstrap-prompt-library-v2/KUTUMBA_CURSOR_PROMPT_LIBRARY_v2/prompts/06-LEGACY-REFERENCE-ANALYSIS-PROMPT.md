# Prompt 06 — Legacy and External Reference Analysis

## Objective

Extract useful knowledge from legacy Bhaktivriksha and devotional reference collections without allowing them to control KUTUMBA or violate copyright.

## Inputs

Use the indexes produced during source ingestion.

Do not reread or copy the entire Granth collection indiscriminately.

## Required analysis

Create:

- `14-research-source-register/LEGACY-BHAKTIVRIKSHA-CROSSWALK.md`
- `14-research-source-register/JAYAPATAKA-SWAMI-BV-MATERIAL-CROSSWALK.md`
- `14-research-source-register/GRANTH-TOPIC-CATALOG.md`
- `14-research-source-register/RESEARCH-BACKLOG.md`
- `14-research-source-register/SOURCE-GAP-MAP.md`
- `17-reviews-and-audits/LEGACY-ADOPTION-REVIEW.md`

Classify every potentially relevant legacy module as:

- adopt
- adapt
- duplicate
- conflict
- archive
- reference only
- rights review required
- doctrinal review required
- worship review required
- safety review required

## Evaluation criteria

Assess:

- alignment with Śrīla Prabhupāda’s teachings
- alignment with current KUTUMBA decisions
- age appropriateness
- family orientation
- practical home application
- source traceability
- copyright and redistribution rights
- worship authorization
- child-protection implications
- overlap with existing KUTUMBA content
- value relative to implementation burden

## No silent import

No legacy lesson, title, sequence, checklist, or worship procedure becomes canonical merely because it already exists.

Every adoption or adaptation must cite:

- source file ID
- source path
- relevant section/page when available
- reason
- modifications
- reviewer required
- rights disposition

## Research rule

Where additional research is needed, prioritize:

1. Śrīla Prabhupāda’s published books and authorized teachings
2. official ISKCON/GBC or temple standards
3. qualified authorized guidance
4. recognized course materials
5. supporting educational research for method only

If internet research is unavailable, create a research ticket. Never fabricate a citation or URL.
